## Descrição

Corrige o `EvidencePanel` no `RiskDashboardV4` que **nunca renderizava** em produção.

### Causa Raiz

O backend (`risk-engine-v4.ts`) armazena `evidence` como objeto `ConsolidatedEvidence`:
```json
{ "gaps": [...], "rag_validated": true, "rag_confidence": 0.8, ... }
```

O frontend `parseEvidence()` esperava `EvidenceItem[]` (array direto). Quando recebia o objeto, `Array.isArray(raw)` retornava `false` e o fallback `JSON.parse(raw as string)` falhava (raw já era objeto), resultando em `[]`.

### Solução

- `parseEvidence` agora detecta objetos `ConsolidatedEvidence` (que possuem `.gaps[]`)
- Mapeia cada gap para `EvidenceItem` com: `fonte`, `prioridade` (gapClassification), `pergunta` (sourceReference), `resposta` (answerValue), `confianca`
- `formatEvidence` (CSV export) também atualizado para o novo formato
- Interfaces `ConsolidatedEvidence` e `ConsolidatedEvidenceGap` adicionadas

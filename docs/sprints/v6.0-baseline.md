## 2. Diagnóstico dos Gaps (Pré-Sprint)

### 2.1 Gap Crítico: ConsistencyGate Desconectado do Fluxo Principal

O `ConsistencyGate.tsx` existe como rota independente (`/projetos/:id/consistencia`) mas **não é chamado pelo `NovoProjeto.tsx`**. O fluxo atual vai diretamente de "Confirmar CNAEs" para a tela de detalhes do projeto, sem passar pelo gate de consistência.

**Impacto:** Projetos com inconsistências críticas (ex: Simples Nacional + faturamento > R$4,8M) avançam sem validação.

### 2.2 Gap de UX: Formulário Frio

O `NovoProjeto.tsx` (1.082 linhas) usa campos `<input>` e `<select>` simples. Não há:
- Score de completude visível
- Sugestões em tempo real
- Perguntas dinâmicas baseadas no perfil
- CTA inteligente ("Avançar" com estado condicional)

### 2.3 Gap de IA: Consistency Engine Sem Análise de Perfil

O `consistencyEngine.ts` (386 linhas) implementa 8 regras determinísticas para regime/porte/faturamento, mas **não analisa a completude e coerência do perfil** antes de gerar as regras. A IA é chamada apenas para findings de consistência, não para sugestões de preenchimento.

### 2.4 Gap de Persistência

Não há persistência do estado intermediário da análise de IA. Se o usuário fechar o modal, todas as sugestões e perguntas dinâmicas são perdidas.

  ├── generateDynamicQuestions() — perguntas contextuais via LLM

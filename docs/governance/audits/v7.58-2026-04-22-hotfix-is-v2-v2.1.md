## Regras operacionais e lições

### Lição v2 (gap de caller)

Gate 0 de hotfix passa a exigir:
> "Verificar caller efetivo em runtime, não apenas caller existente no código."

F3 original do v1.2 rodou `grep -rn "categorizeRisk" server/` mas não
validou qual engine é usado em produção (v3 vs v4). Hotfix aplicado em
engine inativo.

### Lição v2.1 (gap de teste de persist)

Gate 0 de hotfix passa a exigir:
> "Quando hotfix toca valores de schema ENUM ou FK target, exigir teste
> integration que execute INSERT contra DB real (ou schema mock fiel)
> antes do merge."

Os 4 testes Bloco G mockavam `getCategoryByCode` retornando null — FK
constraint só é testada quando `persistRiskV4` executa INSERT real.
Bug de FK escapou local, smoke review Manus, CI inteiro, merge e
deploy — só apareceu no UAT do usuário.

### Lição de label

`Structural Fix Gate` não escuta event `labeled`. Para re-disparar com
labels atualizadas, **empty commit** é a via única sem squash/force-push.
PR Governance workflow **escuta** `labeled` e re-dispara automaticamente.

### Política ADR confirmada

Amendments do ADR-0030 v1.1 entram **inline** no próprio arquivo, sem
criar v1.2 ou v1.3 separados. Hash do arquivo muda progressivamente:
`262f14e1` → `9e89bbfe` (amendment 1 — v2) → `620b0a0b` (amendment 2 — v2.1).

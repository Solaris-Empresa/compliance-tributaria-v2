## Lições aprendidas (para próxima sessão)

1. **Antes de começar reforma de fórmula:** ORQ-20 formal. ADR curto
   vale mais que 10 commits de iteração.

2. **Não confiar em `tsc` para validação visual.** Ambiente deployado
   + screenshot/print É o teste. Se não tenho acesso, o P.O. é QA —
   então ritmo de entrega deve ser 1 fix, 1 validação.

3. **ORQ-19 entre blocos, não apenas no fim.** A auditoria v7.50 no meio
   da sessão foi a que pegou os 2 bugs críticos. Se tivesse feito outra
   antes do UAT pós-V1 completo, teria pego a tabela "895%" antes.

4. **Se P.O. sugere formato visual (ex: "pipes literais com blank lines"),
   usar EXATAMENTE o que ele sugeriu.** Não interpretar/melhorar.
   "Se o user disse X, entregar X." Minha tendência de "fazer melhor"
   virou retrabalho.

5. **Quando P.O. expressa frustração ("já errou 4/5 vezes")**, a resposta
   correta NUNCA é mais código — é pausa + protocolo de disciplina.

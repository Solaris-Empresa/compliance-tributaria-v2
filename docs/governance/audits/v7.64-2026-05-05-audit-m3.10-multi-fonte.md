## Achado central — Falso negativo no DoD original

O DoD inicial da Sprint M3.10 (script `scripts/dod-3780001.ts`, Manus sandbox, **não commitado**) reportou:

```
source_priority=iagen:       2 riscos
source_priority=regulatorio: 6 riscos
→ 2 fontes distintas → PASS (multi-fonte agregado)

evidence.gaps[*].fonte = []  em todos os 8 riscos
→ multi-fonte por risco vazio (Frente 2 não exercitada)
```

A segunda linha era **falso negativo** — artefato de bug no script, não no pipeline. Causa raiz: `JSON.parse(row.evidence)` aplicado a coluna mysql2 já parseada (driver retorna objeto JS, não string).

```typescript
// BUGADO no script (não commitado):
const ev = JSON.parse(row.evidence || '{}');
//   row.evidence === { gaps: [...] }   (objeto)
//   JSON.parse(objeto) → toString() → "[object Object]"
//   → SyntaxError silenciado em try/catch
//   → ev = {} → ev.gaps = undefined → fontes = []
```

Ver Lição #72 em `.claude/rules/governance.md` para o pattern correto e CI gate proposto.

## Conflito RACI residual (observação sistêmica)

Sprint M3.10 expôs limitação da matriz RACI ORQ-33 quando o validador é também o autor da ferramenta de validação. No caso M3.10:

- Manus implementou solaris/iagen analyzers (writers de gaps)
- Manus escreveu o script DoD (medidor da implementação)
- Manus reportou DoD PASS/FAIL (validador)

Em sprints pequenas o overlap é inevitável. A salvaguarda residual é **commitar o script DoD** — assim outro implementador ou o Orquestrador podem re-executar e detectar discrepância. Lição #71 formaliza isso.

Backlog: refinar REGRA-ORQ-33 para o caso "validador-autor-de-ferramenta" — não bloqueante para M3.10.

### Bloco 5 — Correção (CT-D2) — 🟡 **RESSALVA**

**Ação:** botão "Corrigir" → texto: *"Além de combustível, transportamos também óleo vegetal de milho"*

| Verificação | Resultado |
|---|---|
| Versão 3 criada | ✅ |
| "óleo vegetal" mencionado no Resumo | 🟡 **Parcial** — cita "fatos adicionais (óleo vegetal)" de forma genérica, sem elaborar |
| Novo gap específico sobre óleo vegetal | 🔴 **Não criado** — manteve os 4 gaps originais |
| Registrada no histórico | ✅ |

**Análise crítica:**

A correção sobre "combustível" e "óleo vegetal de milho" é **inconsistente com o perfil da empresa** (Distribuidora de Alimentos, CNAE 4639-7/01, não transporta combustível). O LLM detectou o conflito contextual e suavizou a incorporação.

**Hipóteses:**

- **Hipótese A (benigna):** comportamento correto — LLM filtrou contexto inconsistente. A correção é incompatível com o CNAE e NCMs cadastrados (cesta básica).
- **Hipótese B (regressão):** o fix `39aaa13` (regra "cada correção gera pelo menos 1 gap ou oportunidade") pode estar falhando quando a correção é contraditória.

**Teste decisivo pendente:**
Aplicar nova correção **consistente com o perfil**, ex: *"Também comercializamos farinha de trigo NCM 1101.00.10"*.
- Se criar gap/oportunidade → Hipótese A confirmada (CT-D2 🟢)
- Se não criar → Hipótese B confirmada (CT-D2 🔴 regressão)

**Veredito:** 🟡 — funcionamento parcial, precisa teste decisivo antes de fechar.

## Lições da UAT parcial

1. **Fórmula ponderada funciona matematicamente.** O cálculo é reproduzível manualmente e bate com o sistema.

2. **Correção tímida com contexto inconsistente é comportamento aceitável.**
   O LLM age com cautela quando o usuário reporta algo fora do perfil
   declarado. Isso pode ser **feature**, não bug — mas precisa ser documentado
   no prompt oficial (se não estiver).

3. **Complemento RECOF mostra robustez.** Quando o contexto é consistente,
   o sistema cria gap novo + menciona em múltiplas seções, provando que
   o fix `39aaa13` funciona como esperado.

4. **Determinismo é o fix mais importante validado.** O bug original
   (2→1→2 inconsistências em gerações consecutivas) não se reproduz
   mais — validação direta de `temperature=0` no prompt.

## 7. Achados estruturais (auditoria do produto)

### 🔴 Achado crítico: M1 v3 → RAG está DESACOPLADO

Investigação do código (`server/rag-retriever.ts:259-284` + 6 callers em `server/routers-fluxo-v3.ts`):

`retrieveArticles(cnaes, contextQuery, topK)` recebe APENAS:
- Lista de CNAEs (códigos)
- String de contexto livre (description + correction + complement + answersText)

**NÃO recebe** nenhum campo do `PerfilDimensional` produzido pelo M1 v3 (`objeto`, `papel_na_cadeia`, `tipo_de_relacao`, `territorio`, `regime`, `subnatureza_setorial`, `derived_legacy_operation_type`).

**Snapshot do M1** (`buildSnapshot`) é persistido em `m1_runner_logs` (tabela de monitoring). **Nenhum router downstream lê** essa tabela para alimentar RAG/briefing/risk-engine.

**Caminho real** que alimenta briefing/RAG/riscos continua sendo o **legado**: `companyProfile` + `operationProfile` (single-select preenchido pelo `PerfilEmpresaIntelligente.tsx`).

**Implicação:** os 5 casos agro do PR #861 que agora derivam `agronegocio` no M1 não impactam o gate IS em produção — o cliente continua marcando `operationType` manualmente no form legado.

### 🟡 18 casos `setor_regulado=true` derivando papel não-regulado

Apenas S15 (plano de saúde) e S18 (banco digital) derivam `papel=operadora_regulada`. Os outros 18 casos com `setor_regulado=true` (incluindo S24 telecom, S25 energia, I34 farma, I39 refinaria, M52 mineração, T53 tabaco, etc.) derivam `prestador`/`fabricante`/`transportador`. Causa: `derivePapel()` só promove para `operadora_regulada` quando `posicao_na_cadeia_economica="Operadora"` literal.

### 🟡 Chapters NCM ainda não mapeados (14)

A48 (carnes ch.02), A50 (celulose ch.47), e mais 12 chapters presentes na suite mas não em `REGIME_TUPLE_TO_OBJETO_NCM`. Casos continuam derivando fallback INFO.

### 🟡 Corpus RAG: 373 docs ainda tagged "64,65,66"

PR #856 corrigiu 7 docs críticos. PR #860 corrigiu Art. 169 (2 chunks). Mas o padrão estrutural permanece em ~373 documentos no seed (incluindo Art. 3, 5, 6, 7, 11, 12, 13 etc. — fundamentos universais de IBS/CBS). Mitigação parcial: retriever usa `OR` entre keyword e cnae, então não quebra; mas boost por CNAE atrofiado.

## 2. Diagnóstico do erro reportado pelo P.O.

### 2.1 Decodificação do snippet (CSV inline com JSON embutido)

O P.O. enviou texto que começa com `Regime tributário: lucro_presumido` e termina com `,6,1,Uires Tapajós,auto_generation`. Análise:

- **NÃO é JSON válido isolado** — é dump SQL/CSV de uma row de `risks_v4`
- Colunas inferidas (sufixo `,6,1,Uires Tapajós,auto_generation`):
  - `6` = `evidence_count`
  - `1` = `created_by`
  - `Uires Tapajós` = `owner_name` (joined)
  - `auto_generation` = `trigger_type` (`risks-v4.ts:1042,1052`)
- **Conclusão:** matriz foi gerada com sucesso pelo auto-trigger. `trigger_type=auto_generation` confirma o fluxo da Seção 1.3.

### 2.2 Conteúdo do risco — inconsistência tripla detectada

| Campo | Valor reportado | Esperado para perfil agro |
|---|---|---|
| `categoria` (breadcrumb[1]) | `regime_diferenciado` | Plausível (agro → keyword match em risk-categorizer.ts:94) |
| `artigo` (breadcrumb[2]) | `Art. 126 LC 214/2025; Arts. 200-245 Decreto 12.955/2026` | **Art. 164 LC 214/2025** (produtor rural) |
| `rag_artigo_exato` | `Art. 204` | **Art. 164** ou `Art. 126` (mas não Art. 204 que é consórcio) |
| `rag_query` | `"regime diferenciado"` | Query genérica demais — captura Art. 204 (consórcio fala "regime diferenciado") |
| `rag_trecho_legal` | "Art. 204. Na administração de consórcio…" | Trecho de produtor rural ou regime agro |
| `source_priority` | `regulatorio` | Esperado (não há gap SOLARIS contribuindo aqui) |
| `gap_detected` | `false` | Risco inerente ao arquétipo, não detectado por questionário |

### 2.3 Causa raiz B1 — Categorização keyword-based

**Arquivo:** `server/lib/risk-categorizer.ts:89-99`

```ts
// ── 2. Regime Diferenciado (saúde, educação, agro, financeiro) ──
if (
  topicos.includes("regime diferenciado") || topicos.includes("específico") ||
  desc.includes("saúde") || desc.includes("medicamento") || desc.includes("farmacêutico") ||
  desc.includes("educação") || desc.includes("ensino") ||
  desc.includes("agropecuário") || desc.includes("agro") || desc.includes("rural") ||
  desc.includes("serviço financeiro") || desc.includes("seguro") ||
  desc.includes("regime específico") || desc.includes("regime diferenciado")
) {
  return "regime_diferenciado";
}
```

**Problema:** uma única categoria `regime_diferenciado` agrupa saúde + educação + agro + financeiro + consórcio. O sistema não distingue qual sub-domínio está em jogo — qualquer keyword agro/rural cai aqui.
